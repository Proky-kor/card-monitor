# GitHub로 완전 자동화 (내 PC 없이 클라우드에서)

## 이게 뭔가요
GitHub의 무료 기능(Actions + Pages)으로 **내 PC를 안 켜도** 매일 자동으로:
1. 8개 카드사 점검 → 2. 신규 상품 생기면 **Teams 알림** → 3. **웹 대시보드(무료 호스팅)** 자동 갱신.

## PC 작업 스케줄러 vs GitHub — 솔직 비교
| | 내 PC(작업 스케줄러) | GitHub (클라우드) |
|---|---|---|
| 설정 난이도 | 쉬움(.bat 더블클릭) | 보통(계정·업로드·Secret 한 번) |
| PC 켜둬야? | **예** | 아니오 |
| 대시보드 | 내 PC에서만 | **무료 공개 URL** |
| 비용 | 0 | 0 (공개 저장소) |

> **단 하나의 불확실성**: 한국 카드사 사이트가 GitHub(해외) 서버 IP를 막을 수 있습니다. 막히면 클라우드 수집이 실패하므로, **설정 후 첫 수동 실행으로 꼭 확인**하세요. 막히면 PC 방식으로 쓰면 됩니다.

---

## 설정 (한 번만, 약 15분)

### 1) GitHub 계정 + 저장소
1. https://github.com 가입(무료).
2. 우측 상단 **+** → **New repository** → 이름(예: `card-monitor`) → **Public** 선택 → Create.
   - Public이어야 Pages가 무료입니다. **민감정보(웹훅)는 코드가 아니라 Secret에 넣어 안전**합니다.

### 2) 파일 업로드
저장소 첫 화면 → **uploading an existing file** 클릭 → 이 폴더에서 **아래만** 끌어다 놓기:
- 폴더: `data`, `services`, `storage`(빈 폴더면 생략 가능), `web`, `.github`
- 파일: `main.py`, `config.py`, `requirements.txt`, `pyproject.toml`
- ❌ 올리지 마세요: `.venv`, `_probe`, `dist`, `.env`, `storage/cards.db`, 이미지들
  (용량 크거나 비밀이거나 자동 생성됩니다.)
- Commit changes.

### 3) Teams 웹훅 등록 (알림 받으려면)
1. 저장소 **Settings → Secrets and variables → Actions → New repository secret**
2. Name: `CARD_TEAMS_WEBHOOK_URL`, Secret: (Teams 웹훅 URL 붙여넣기) → Add secret.
   - 웹훅 URL 만드는 법은 `자동화_안내.md`의 Teams 항목 참고.
   - ※ Secret은 안전하게 저장됩니다(코드/로그에 노출 안 됨).

### 4) Pages 켜기 (웹 대시보드)
**Settings → Pages → Build and deployment → Source: GitHub Actions** 선택.

### 5) 첫 실행(테스트)
**Actions 탭 → "카드 모니터링..." → Run workflow → Run workflow** 클릭.
- 5~10분 후 초록 체크면 성공. 빨간 X면 로그 확인(대개 한국 사이트 IP 차단 → 이땐 PC 방식 사용).
- 성공 시: Settings → Pages 의 URL(`https://아이디.github.io/card-monitor`)에서 대시보드 확인.

## 그 다음은 전자동
- 매일 09:00(한국시간) 자동 점검 → 신규 시 Teams 알림 → 대시보드 자동 갱신.
- 시각 변경: `.github/workflows/monitor.yml`의 `cron: "0 0 * * *"` (UTC 기준, 00:00 UTC=09:00 KST).
- 수동 점검: 언제든 Actions 탭에서 Run workflow.

## 동작 원리(참고)
- 상태(어떤 카드가 있었는지)는 `storage/cards.db`로 저장소에 자동 커밋되어, 다음 실행 때 **새로 생긴 카드만** 신규로 잡습니다.
- 대시보드 이미지는 카드사 원본 URL을 직접 보여줘 저장 공간이 들지 않습니다.
- 비밀값은 Secret에만, 코드에는 없습니다.
